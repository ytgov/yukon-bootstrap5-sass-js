Make your service look like Yukon.ca.

Start with this template to include our base styles in your application code. Add or remove components as needed.

CSS styles used in the design system, including colours and sizes are not to be modified.

See the Digital Service Delivery Guide including the Online design patterns section for more details on how to use these components.

This version of the Yukon.ca Design System is based on Bootstrap 5. See the Bootstrap 5 Sass & JS example documentation for more details.

Alerts

Use alerts to provide contextual feedback messages for typical user actions. Choose from a handful of available and flexible alert messages.

You can wrap any text and an optional dismiss button in .alert, or one of the four contextual classes (for example, .alert-success) for basic alert messages.

Alerts don't have default classes, only base and modifier classes. A default gray alert doesn't make too much sense, so always specify a type with a contextual class. Choose from success, info, warning, or danger.

See the Bootstrap 5 guidance for alerts.

Success alert

Use this alert to call attention to information that needs to stand out from the rest of the page content. With the Success alert, the title appears next to a a checkmark icon; the content background is green.

An example success alert with an icon

Information alert

Use to highlight targeted information for specific segment groups. With the Info alert, the title appears next to a letter "i" icon; the content background is blue. Use this alert to call attention to feature-worthy information that needs to stand out from the rest of the page content.

An example alert with an icon

Warning alert

Use this alert to call attention to information that needs to stand out from the rest of the page content. With the Warning alert, the title appears next to a caution icon; the content background is yellow.

An example warning alert with an icon

Danger alert

Use this alert to call attention to information that needs to stand out from the rest of the page content. With the Danger alert, the title appears next to a caution icon; the content background is red.

An example danger alert with an icon

Buttons

Use buttons to create a call to action that entices the user to submit content from a web form. You can also use buttons to send users to another online service that provides additional information.

Write button text in sentence case, describing the action it performs. Text usually follows a verb + noun pattern.

Buttons to move to forward and backward steps in a form or flow should follow reading order (backward/previous on the left, and forward/next on the right).

Align buttons to the left edge of your form.

See the Bootstrap 5 guidance for buttons, buttons groups, and dropdowns.

Disabled buttons

Disabled buttons have poor contrast and can confuse some users, so avoid them if possible.

Code (to update)
<div class="form-actions webform-actions form-wrapper">
	<button class="webform-button form-submit btn-default btn" type="submit" id="" name="" value="Go back a step">Go back a step</button>
	<button class="webform-button btn-primary form-submit btn" type="submit" id="" name="" value="Continue to next step">Continue to next step</button>
</div>

Tables

Use to organize data into rows and columns in order to create data cells.

Add the base .table CSS to any table. Given the widespread use of tables for other plugins like calendars and date pickers, here we list some isolated custom table styles.

Use .table-striped to add zebra-striping to any table row within the <tbody>.

Add .table-hover to enable a hover state on table rows within a <tbody>. In most cases, .table-striped is preferred to .table-hover.
